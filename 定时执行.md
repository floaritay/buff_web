# 电脑端自动定时执行脚本

## .bat脚本
创建一个txt文件，将下面内容复制进去，保存为.bat文件
```bash
@echo off
:: 你的主脚本内容（例如：运行BUFF饰品购买脚本）
python buff_buyer.py 
:: pause :: 暂停脚本执行，等待用户确认
:: 等待 2 秒后自动关闭窗口（不显示倒计时）
timeout /t 3 >nul
exit ::退出当前窗口
```

## 使用方法
点击.bat文件即可运行脚本

## 定时执行
打开任务计划程序（命令行运行 taskschd.msc 或 win+s 搜索 任务计划程序）
创建一个任务：
1. 点击“创建任务” (注意不是“创建基本任务”)
2. 在“常规”选项卡中，填写任务名称与描述（例如：buff_buyer）
3. 点击“触发器”选项，点击“新建”添加一个新的触发器，在“高级设置”中选择“重复任务间隔”设置为“5分钟”，持续时间设置为“无期限”。点击“确定”退出
4. 点击“操作”选项，点击“新建”，选择“启动程序”，在“程序或脚本”处点击“浏览”，找到并选择.bat文件，点击“确定”退出
5. 在“条件”选项卡中，勾选“唤醒计算机以运行此任务”。需要的话还可以设置电池状态（例如：仅在交流电源时运行）。点击“确定”退出
6. 点击“确定”保存任务

现在任务计划程序会自动运行.bat文件，并每隔5分钟重复执行一次。不再需要手动点击.bat文件运行脚本。
注意：需保持电脑开机。在某些情况下，电脑长时间未动等可能自动关机，需要修改电脑的“电源计划”。具体修改方式自己搜索。

# 手机端（安卓）自动定时执行
下载termux应用（在Google Play商店，浏览器或F-Droid中搜索安装）

## 配置环境
打开 Termux，执行以下命令配置 Python 环境
```bash
# 更新软件源
pkg update && pkg upgrade -y
# 安装Python（3.x版本）
pkg install python -y
# 用到的第三方库，需提前安装
pip install requests
#  Termux 没有访问安卓公共存储（Download 文件夹）的权限
# 执行权限授予命令
termux-setup-storage
# 执行后会弹出安卓系统的权限请求弹窗，一定要点击 “允许”，否则无法访问存储。
# 执行成功后，Termux 会在~（主目录）下创建一个storage软链接，指向安卓的公共存储，路径对应关系：
# 手机 Download 文件夹 → Termux 中~/storage/downloads
```

## 脚本
如果你想像 bat 一样双击 / 一键执行，需要把.bat 逻辑改写成安卓支持的 shell 脚本
新建run_buff.sh文件，内容如下：
```bash
#!/bin/bash
# 切换到脚本目录（根据你手机实际存放路径修改）
cd storage/downloads/Buff
# 执行核心脚本
echo "开始运行buff_charm_searcher.py..."
python  buff_buyer.py

# 模拟bat里的pause，按回车退出
# read -p "脚本执行完成，按回车键退出..."
```

给.sh 脚本加权限并运行
```bash
# 进入Download目录
cd /storage/downloads/Buff
# 添加可执行权限
chmod +x run_buff.sh
# 运行脚本
./run_buff.sh
# 运行脚本
./run_buff.sh
```

## 定时执行
1. 安装cronie  
```bash
# 安装cronie（crontab的核心包）
pkg install cronie -y
# 启动cron服务
crond start
# 查看crond服务状态
crond status
# 或通过进程查看
pgrep crond
# 如果运行正常，pgrep crond会输出一个数字（crond 的进程 ID），crond status会显示crond is running。

# 如果希望 Termux 每次启动时自动运行 crond：
# 配置自启
echo "crond start" >> ~/.bashrc
# 生效配置
source ~/.bashrc
```

2. 编辑定时任务  
```bash
# 编辑定时任务，打开编辑器（默认是 nano）
crontab -e
```
在编辑器末尾添加以下内容
```bash
# 每5分钟执行一次Python脚本，并记录日志（方便排查问题）
*/5 * * * * python ~/storage/downloads/Buff/buff_buyer.py >> ~/buff_buyer.log 2>&1
# 按 Ctrl + O 保存修改；
# 按回车键确认文件名；
# 按 Ctrl + X 退出编辑器。
```

验证定时任务是否生效  
等待 5 分钟后，查看日志文件确认脚本是否执行：
```bash
# 查看日志内容
cat ~/buff_buyer.log
```

没问题后可以去掉日志
```bash
crontab -e
# 在编辑器末尾添加以下内容
*/5 * * * * python ~/storage/downloads/Buff/buff_buyer.py
# 按 Ctrl + O 保存修改；
# 按回车键确认文件名；
# 按 Ctrl + X 退出编辑器。
rm ~/buff_buyer.log
```


其他命令
```bash
# 停止服务
crond stop
# 重启服务
crond restart
# 查看当前定时任务
crontab -l
# 编辑定时任务
crontab -e
# 删除所有定时任务
crontab -r
```
## Cron表达式说明

标准的 cron 表达式格式为：`分 时 日 月 周`

| 字段 | 允许值 | 特殊字符 |
|------|--------|----------|
| 分 | 0-59 | , - * / |
| 时 | 0-23 | , - * / |
| 日 | 1-31 | , - * / ? L W |
| 月 | 1-12 或 JAN-DEC | , - * / |
| 周 | 0-6 或 SUN-SAT（0=周日） | , - * / ? L # |

常用示例：
- `0 * * * *` - 每小时执行
- `0 */2 * * *` - 每2小时执行
- `0 9 * * *` - 每天9点执行
- `0 18 * * *` - 每天18点执行
- `0 */30 * * *` - 每30分钟执行
- `30 8 * * 1` - 每周一8点30分执行
